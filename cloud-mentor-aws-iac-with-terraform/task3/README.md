# Cloud-Mentor: AWS IaC with Terraform — Task 3: S3 Storage

## Overview

Standalone Terraform configuration provisioning an S3 bucket in AWS. Written as part
of the **Cloud-Mentor: AWS IaC with Terraform** instructor-led sandbox course.

The sandbox is ephemeral — credentials are pre-generated by the course platform via
CloudFormation and all resources are automatically destroyed at session end.

## File Structure

```
.
├── main.tf            # Provider configuration
├── versions.tf        # Terraform and provider version constraints
├── variables.tf       # All variable declarations
├── storage.tf         # S3 bucket resource
├── outputs.tf         # Output values (bucket name, ARN)
├── terraform.tfvars   # Variable values (sandbox-safe — see below)
└── README.md
```

---

## Why `.tfvars` Is Committed

All values in `terraform.tfvars` are sandbox-scoped — ephemeral IDs and non-sensitive
configuration that expire with the session. No production credentials or secrets are
stored here.

> **Do not replicate this in real-world projects.** Sensitive `.tfvars` files must be
> excluded via `.gitignore` and managed via a secrets manager or CI/CD variable store.

## Usage

```bash
# 1. Initialise working directory
terraform init

# 2. Format and validate
terraform fmt
terraform validate

# 3. Preview changes
terraform plan

# 4. Deploy
terraform apply
```

---

## Outputs

After `terraform apply`, the following values are printed:

| Output | Description |
|---|---|
| `bucket_name` | Name of the created S3 bucket |
| `bucket_arn` | ARN of the created S3 bucket |

---

## Verification

After `terraform apply`:

- Check the [AWS S3 console](https://s3.console.aws.amazon.com/s3/buckets) to confirm
  the bucket `cmtr-znhvv6bs-bucket-1776165757` exists
- Confirm the `Project=cmtr-znhvv6bs` tag is applied under the bucket's **Properties** tab
- Review `terraform plan` and `terraform apply` output for errors
