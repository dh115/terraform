# Cloud-Mentor: AWS IaC with Terraform — Sandbox Course

## Overview

This repository contains Terraform code written as part of the **Cloud-Mentor: AWS IaC with Terraform** instructor-led sandbox course. The course covers Infrastructure as Code fundamentals and AWS resource provisioning using Terraform in a fully managed, ephemeral sandbox environment.

The sandbox is pre-configured with temporary AWS credentials, pre-generated environment variables, and scoped IAM permissions. **All infrastructure is automatically destroyed when the session ends.** Nothing here is intended for production use.

---

## Repository Purpose

This repo serves as a working record of the course exercises — tracking progress, preserving configuration patterns, and demonstrating hands-on Terraform skills. Exercises may include:

- Provider configuration and backend setup
- VPCs, subnets, route tables, and security groups
- EC2 instances, IAM roles, and S3 buckets
- Remote state management
- Modularization and variable management

---

## Why `.tfvars` Files Are Committed to This Repository

Normally, `.tfvars` files are excluded from version control via `.gitignore` because they often contain sensitive values such as account IDs, access keys, or secrets.

**This repository is an exception for the following reasons:**

1. **Sandbox credentials are ephemeral.** All AWS credentials and environment values are auto-generated by the course platform and expire when the session ends. They carry zero long-term risk if exposed.

2. **No real secrets are stored.** The `.tfvars` values (region, sandbox account ID, resource naming, etc.) are scoped to a throwaway environment with no access to production systems.

3. **Reproducibility matters for learning.** Committing `.tfvars` alongside `.tf` files means each exercise snapshot is complete and self-contained — making it easy to review, diff, and learn from later.

4. **Collaboration and review.** In a course context, instructors or peers may need to see the full configuration to provide feedback. Excluding variables would make that harder.

> **Important:** Do not replicate this practice in real-world or production repositories. Sensitive `.tfvars` files must always be added to `.gitignore` and managed via secrets managers, CI/CD variable stores, or encrypted backends.

## Prerequisites

- Terraform >= 1.0
- AWS credentials injected via the sandbox environment (do not configure manually)
- Course sandbox session active

---

## Usage

```bash
terraform init
terraform plan
terraform apply
```

Destroy is handled automatically by the sandbox platform at session end, but can also be run manually:

```bash
terraform destroy
```
